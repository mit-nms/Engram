## STRUGGLE PROTOCOL

You are STUCK when: same approach fails to improve score after 3 attempts. **Being stuck does NOT mean stop.** It means: diagnose, then try a different approach and run more simulations. You are the only agent — use the struggle protocol to unstick yourself and keep making progress.

### If Simulation Fails (timeout, infeasible, error)

1. If it is an obvious error to address, fix it and run simulation again.

2. If you are using an optimization:
   - **DO NOT immediately switch to a heuristic**
   - Check what's wrong:
   -- Status "Infeasible" → constraints too tight, try relaxing one
   -- Status "Not Solved" / timeout → problem too big, reduce graph size
   -- Status "Optimal" but bad score → formulation is wrong, check objective

3. Add debug prints to identify the issue:
   ```python
   print("SOLVER STATUS:", prob.status)
   print("OBJECTIVE VALUE:", value(prob.objective))
   print("NUM VARIABLES:", len(prob.variables()))
   ```

4. Try ONE fix at a time and test. Run more simulations after each fix.
5. Only abandon this approach after you understand WHY it fails. Then try a different approach (e.g. approximation, smaller subproblem, different formulation) and keep running simulations.

### If Score Plateaus (no improvement for 3 runs)

1. Look at per-configuration results — which config is worst?
2. Focus optimization on the worst-performing configuration; run simulations to test.
3. Check if you're hitting a constraint boundary (all capacity used?)
4. If minor changes aren't helping, try a fundamentally different formulation and run more simulations.
5. **Do not stop.** Plateau on one approach means switch approach and keep going.

### If Same Approach Fails 3+ Times

A "failure" = score doesn't improve or gets worse.

1. Document what you tried and your hypothesis for why it failed.
2. Choose a completely different approach (not a variation).
3. **Run more simulations** with the new approach. You are the only agent — keep experimenting until the system stops you.

### If Simulation Times Out (CRITICAL)

**DO NOT just abandon the approach.** Timeouts mean the algorithm is too slow, not that the idea is wrong.

1. **Add progress logging** to understand WHERE it's slow:
   ```python
   import time
   start = time.time()
   print(f"Step 1")
   # ... your code ...
   print(f"Step 1 done in {time.time() - start:.2f}s")
   
   print(f"Step 2")
   # ... rest of code ...
   print(f"Step 2 done in {time.time() - start:.2f}s")
   ```

2. **Log intermediate state** so you can inspect what happened:
   ```python
   print(f"variables: {variables}")
   ```

3. Try making the problem tractable (smaller graph, time limit, relaxation) and run more simulations. Do not stop — keep iterating.
